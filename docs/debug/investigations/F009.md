## Issue

**F009 — emFileLinkPanel shows dark-red 'No file model' before AutoExpand**

When a cosmos item first appears (before the user zooms in enough to trigger AutoExpand on the emFileLinkPanel), the panel renders a dark-red 'No file model' status. Root cause: `emFileLinkPanel.file_panel` (an `emFilePanel`) is never connected to the `emFileLinkModel` via `SetFileModel` because `emRecFileModel` does not implement the `FileModelState` trait. In C++, `emFilePanel::SetFileModel` would be called with the model, making the loading state show as 'Wait...' (olive) or disappear once loaded. The fix requires `emRecFileModel` to implement `FileModelState`, or a wrapper that does.

Repro: Launch app. Navigate into cosmos. Observe file manager panels (FS, Home) — they show a small dark-red box before zooming in. Once zoomed in enough for AutoExpand, the red disappears.

## Phase 1 — Root Cause Investigation

**E1**: C++ `emFileLinkPanel` constructor (line 31): `: emFilePanel(parent, name, fileModel, true)` — model is passed to `emFilePanel` base constructor, which calls `SetFileModel` internally. The C++ panel is never in `NoFileModel` state after construction.

**E2**: Rust `emFileLinkPanel::set_link_model` (emFileLinkPanel.rs:87) only stores `self.model = Some(model)`. `file_panel.SetFileModel` is never called. `file_panel` remains at `last_vir_file_state = NoFileModel`.

**E3**: `emFilePanel::Paint` guard (emFileLinkPanel.rs:209): `!self.file_panel.GetVirFileState().is_good() && self.child_panel.is_none()` — with `NoFileModel`, `is_good()` = false. Before AutoExpand creates `child_panel`, this fires → `paint_status` renders `rgba(128, 0, 0, 192)`.

**E4**: `emRecFileModel<T>` does not implement `FileModelState` (confirmed by grep). It has all required data fields (`state: FileState`, `error_text: String`, `memory_need: u64`) but `GetFileStateSignal()` has no backing `SignalId` field. `emFilePanel::SetFileModel` does not call `GetFileStateSignal()` → returning `SignalId::default()` is safe.

**E5**: Failing test written and confirmed: `set_link_model_connects_file_panel` in `emFileLinkPanel::tests` — failed before fix (`NoFileModel`), passes after (`Waiting`).

## Phase 2 — Pattern Analysis

**C++ reference**: `emFileLinkPanel(parent, name, fileModel, true)` — the `emFilePanel` base constructor receives the model pointer at construction. In C++, `emFilePanel` owns the model reference and transitions VirtualFileState from `Waiting` → `Loaded` as the model loads. Never hits `NoFileModel`.

**Rust gap**: `emRecFileModel<T>` is a standalone port ("Does not wrap emFileModel<T>") with no `FileModelState` impl. This was a deliberate simplification to avoid self-referential borrow issues, but the consequence is that `emFilePanel` can't be connected to `emRecFileModel`-backed models.

**Secondary**: `emFileLinkModel` wraps `emRecFileModel<emFileLinkData>`. Implementing `FileModelState` on both via delegation is clean and requires no new fields.

## Phase 3 — Hypotheses

HYPOTHESIS: Implementing `FileModelState` for `emRecFileModel<T>` (and delegating from `emFileLinkModel`) and calling `file_panel.SetFileModel` in `set_link_model` changes the initial VirtualFileState from `NoFileModel` to `Waiting`, eliminating the dark-red render. / OUTCOME: CONFIRMED / EVIDENCE: E5 — test `set_link_model_connects_file_panel` passes after fix; 2686 tests pass, 0 fail.

## Root Cause

`emFileLinkPanel::set_link_model` never called `file_panel.SetFileModel`, leaving `file_panel.last_vir_file_state = NoFileModel`. Before `AutoExpand` creates the child panel, `emFileLinkPanel::Paint` hit the `!is_good() && child_panel.is_none()` guard and called `paint_status(NoFileModel)` → `rgba(128, 0, 0, 192)` dark-red render.

**Fix** (three files):
1. `emcore/src/emRecFileModel.rs`: `impl<T: Record + Default> FileModelState for emRecFileModel<T>` — delegates to existing fields; `GetFileStateSignal()` returns `SignalId::default()` (safe: `emFilePanel::SetFileModel` does not use it).
2. `emfileman/src/emFileLinkModel.rs`: `impl FileModelState for emFileLinkModel` — delegates to `rec_model`.
3. `emfileman/src/emFileLinkPanel.rs`: `set_link_model` calls `file_panel.SetFileModel(Some(Rc::clone(&model) as Rc<RefCell<dyn FileModelState>>))` before storing `self.model`.
