## Task 1 — W5a: Rewrite Phase-8 close-signal test as single-engine drive loop

**Files:**
- Modify: `crates/emcore/src/emView.rs:6104-6185` (function `test_phase8_popup_close_signal_zooms_out`)

**Context:** The current test (a) verifies wake-up with a real `WakeUpUpdateEngineClass`, then (b) swaps in a `NoopEngine`, disconnects the signal from both engines, manually calls `sched.fire(close_sig)`, drives `DoTimeSlice` so `sig.clock` advances past the dummy's clock, and finally calls `v.Update()` directly. The reviewer's complaint is the two-engine hop.

**Implementation insight (added after initial implementer dispatch):** The old test's `NoopEngine` swap and direct `v.Update()` call were papering over a real production constraint — `UpdateEngineClass::Cycle` dispatches through `ctx.windows.get(&self.window_id)`, so a bare `emView` attached to the scheduler with `WindowId::dummy()` and no window in the registry cannot receive `Update` via the engine. This is a consequence of the Rust ownership choice (`emWindow.view: emView` by value rather than `Rc<RefCell<emView>>`), not a C++ divergence — in production every real view lives in a registered `emWindow`. To exercise the real chain in a test, the test must wrap the view in a real `emWindow` (via `emWindow::new_popup_pending`, which constructs without winit/GPU — see `emWindow.rs::new_popup_pending_constructs_without_event_loop` test at line 1721) and register it in the `windows` HashMap at the same id it attached the scheduler with. The `fire(close_sig)` call itself is NOT a hack — `emWindow::SignalClosing` returns the signal ID and the real consumer (the App) calls `scheduler.fire(...)` exactly this way (see `emWindow.rs:1472-1480`). What we're removing is the `NoopEngine` swap and the direct `v.Update()` call.

Replace the body of `test_phase8_popup_close_signal_zooms_out` (the whole function + its `///` doc comment) with the version below. This constructs a real `emWindow` via `new_popup_pending`, registers it in the `windows` HashMap at `WindowId::dummy()`, attaches the scheduler to the view *inside that window*, and then drives the real engine chain. All view mutations go through `main_win.borrow_mut().view_mut()`; care is taken to drop each borrow before `DoTimeSlice` runs (since `DoTimeSlice` will itself `borrow_mut()` the window through the engine's `ctx.windows` lookup).

```rust
/// Phase 8 acceptance test — single-engine integrated run.
///
/// Drives the full real chain: `sched.fire(close_signal)` (the real
/// consumer API per `emWindow::SignalClosing`) wakes the
/// `WakeUpUpdateEngineClass` registered by `attach_to_scheduler`,
/// which cycles `emView::Update` via the window registry, which sees
/// the signaled `PopupWindow->close_signal` and calls `ZoomOut`,
/// tearing the popup down. The test asserts tear-down occurs within a
/// bounded number of `DoTimeSlice` iterations.
///
/// This supersedes the earlier two-engine shape (with a dummy
/// `NoopEngine` swap) that only verified `sig.clock > eng.clock`
/// arithmetic rather than the real wake-up chain. The shape here
/// requires wrapping the view in a real `emWindow` because
/// `UpdateEngineClass::Cycle` dispatches through the window registry;
/// `new_popup_pending` provides a winit/GPU-free constructor suitable
/// for tests (see `emWindow.rs::new_popup_pending_constructs_without_event_loop`).
#[test]
fn test_phase8_popup_close_signal_zooms_out() {
    use crate::emWindow::{emWindow, WindowFlags};

    let (mut tree, root, child_a, _) = setup_tree();
    let sched = Rc::new(RefCell::new(EngineScheduler::new()));

    // Build a real emWindow to host the view. new_popup_pending is the
    // non-materialized constructor; it takes scheduler-allocated
    // signals and does not touch winit/GPU.
    let close_sig_main = sched.borrow_mut().create_signal();
    let flags_sig_main = sched.borrow_mut().create_signal();
    let focus_sig_main = sched.borrow_mut().create_signal();
    let geom_sig_main = sched.borrow_mut().create_signal();
    let bg = crate::emColor::emColor::rgba(0, 0, 0, 0xFF);
    let main_win = emWindow::new_popup_pending(
        root,
        WindowFlags::empty(),
        "phase8-test-main".to_string(),
        close_sig_main,
        flags_sig_main,
        focus_sig_main,
        geom_sig_main,
        bg,
    );

    // Give the main view the same geometry the bare-view test used, and
    // register it + attach its update engine under WindowId::dummy().
    let main_id = winit::window::WindowId::dummy();
    {
        let mut mw = main_win.borrow_mut();
        let view = mw.view_mut();
        view.SetGeometry(&mut tree, 0.0, 0.0, 640.0, 480.0, 1.0);
        view.attach_to_scheduler(sched.clone(), main_id);
    }
    let mut windows: std::collections::HashMap<
        winit::window::WindowId,
        Rc<RefCell<emWindow>>,
    > = std::collections::HashMap::new();
    windows.insert(main_id, main_win.clone());

    // Clear zoomed_out_before_sg, enable popup zoom, and create a popup.
    {
        let mut mw = main_win.borrow_mut();
        let view = mw.view_mut();
        view.Update(&mut tree);
        view.SetViewFlags(ViewFlags::POPUP_ZOOM, &mut tree);
        view.RawVisit(&mut tree, child_a, 0.0, 0.0, 0.1, true);
        assert!(
            view.PopupWindow.is_some(),
            "popup should be created by RawVisit under POPUP_ZOOM"
        );
    }

    // SwapViewPorts (Step 2 of RawVisit) allocates close_signal and
    // connects it to the update engine. Verify the wake-up wire.
    let (close_sig, eng_id) = {
        let mw = main_win.borrow();
        let view = mw.view();
        let cs = view
            .PopupWindow
            .as_ref()
            .map(|p| p.borrow().close_signal)
            .expect("PopupWindow present when scheduler is attached");
        let eid = view.update_engine_id.expect("update engine registered");
        (cs, eid)
    };
    assert!(
        sched.borrow().get_signal_refs(close_sig, eng_id) >= 1,
        "close_signal must be connected to the update engine"
    );

    // Trigger the real consumer API: fire the close_signal (same call
    // shape real App code uses after emWindow::SignalClosing).
    sched.borrow_mut().fire(close_sig);

    // Drive the real scheduler until the popup tears down. Budget is
    // small because the chain is one wake-up + one Update cycle; the
    // loop exists so an unexpected extra tick doesn't flake the test.
    // All test-held borrows on main_win must be dropped by now —
    // DoTimeSlice will borrow_mut() main_win via ctx.windows lookup.
    const MAX_SLICES: usize = 8;
    let mut slices_used = 0;
    let mut torn_down_early = false;
    for i in 0..MAX_SLICES {
        sched.borrow_mut().DoTimeSlice(&mut tree, &mut windows);
        if main_win.borrow().view().PopupWindow.is_none() {
            torn_down_early = true;
            slices_used = i + 1;
            break;
        }
    }
    assert!(
        torn_down_early,
        "popup should tear down within {} DoTimeSlice iterations; \
         still present after exhausting budget",
        MAX_SLICES
    );
    assert!(
        main_win.borrow().view().PopupWindow.is_none(),
        "popup should be torn down after ZoomOut (slices_used={})",
        slices_used
    );

    // Scheduler cleanup for Drop debug_asserts. Reach through the
    // window to clear the view's registered engines/signals.
    {
        let mut mw = main_win.borrow_mut();
        let view = mw.view_mut();
        if let Some(id) = view.update_engine_id.take() {
            sched.borrow_mut().remove_engine(id);
        }
        if let Some(eoi) = view.EOISignal.take() {
            sched.borrow_mut().remove_signal(eoi);
        }
        if let Some(id) = view.eoi_engine_id.take() {
            sched.borrow_mut().remove_engine(id);
        }
        if let Some(id) = view.visiting_va_engine_id.take() {
            sched.borrow_mut().remove_engine(id);
        }
    }
    // Free the main-win signals we allocated above.
    sched.borrow_mut().remove_signal(close_sig_main);
    sched.borrow_mut().remove_signal(flags_sig_main);
    sched.borrow_mut().remove_signal(focus_sig_main);
    sched.borrow_mut().remove_signal(geom_sig_main);
}
```

Notes for the implementer:

- `new_popup_pending` constructs the inner `emView` via `emView::new(root_panel, 1.0, 1.0)` — the root is already set; don't try to re-set it. `SetGeometry(&mut tree, 0.0, 0.0, 640.0, 480.0, 1.0)` is needed because the old test used `emView::new(root, 640.0, 480.0)` which sets the initial viewport size to 640×480; `new_popup_pending` uses 1.0×1.0 placeholder geometry.
- The old test imported `use crate::emEngine::{emEngine as EngineTrait, EngineCtx, Priority};` to define `NoopEngine`. That import is no longer needed — removing the function body removes the import scope too.
- Do NOT add any `#[allow(...)]` attribute.

    ///
    /// DIVERGED: C++ uses a `PanelRingNode* NoticeList` sentinel with raw
    /// pointer linkage (emView.h:576, emPanel.h:823). Rust uses two

    /// with queued notices form a doubly-linked list; `add_to_notice_list`

    /// Divergence classification per the Port Ideology (CLAUDE.md):
    /// - **Storage shape** (global `PanelTree` owns NoticeList vs C++
    ///   per-view `emView::NoticeList`): *Forced.* Rust ownership rules
    ///   make per-view intrusive ring nodes impractical without
    ///   `unsafe` and custom allocators.
    /// - **Data structure** (`Vec`/arena-index vs `PanelRingNode*`
    ///   sentinel): *Idiom adaptation.* Below the observable surface.
    /// - **Dispatch driver** (global call from
    ///   `emGUIFramework::about_to_wait` vs per-view call from
    ///   `emView::Update`, emView.cpp:1312): *Design-intent violation.*

    ///   once globally with a single arbitrarily-chosen pixel_tallness
    ///   (emGUIFramework.rs `pixel_tallness` site). This is the
    ///   load-bearing part of the divergence and is tracked as the
    ///   successor workstream **"Per-view notice dispatch
    ///   (emView.cpp:1312 parity)"** in the emView subsystem closeout
