**Path taken:** Step 3 (no code changes).

Constructors found in `crates/emcore/src/emEngineCtx.rs` (lines 288–313):

```rust
// Constructor without scheduler — wakeup methods are no-ops.
pub fn new(
    tree: &'a mut PanelTree,
    id: PanelId,
    current_pixel_tallness: f64,
) -> Self

// Constructor with scheduler — wakeup methods propagate to the engine.
pub fn with_scheduler(
    tree: &'a mut PanelTree,
    id: PanelId,
    current_pixel_tallness: f64,
    scheduler: &'a mut EngineScheduler,
) -> Self
```

`scheduler` is taken as `&'a mut EngineScheduler` (not `Option<...>`); the `Option` wrapping is internal to the struct field. `PanelCtx::new` sets `scheduler: None`.

Task 2 will use `with_scheduler` at the input-dispatch call site (where a real scheduler is available) and `new` in test-mode contexts where no scheduler exists.
